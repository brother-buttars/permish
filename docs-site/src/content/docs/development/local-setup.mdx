Run Permish locally without Docker for development. This gives you hot module replacement and fast iteration.

## Prerequisites

| Tool    | Version | Install                                      |
|---------|---------|----------------------------------------------|
| Node.js | 24 LTS  | `nvm install` (reads `.nvmrc` automatically) |
| pnpm    | 10+     | `corepack enable && corepack prepare pnpm@latest --activate` |
| nvm     | any     | [github.com/nvm-sh/nvm](https://github.com/nvm-sh/nvm) |

## Clone and Install

```bash
git clone https://github.com/brother-buttars/permish.git
cd permish
```

Set up the environment file:

```bash
cp .env.example .env
# Edit .env with your JWT_SECRET and optional email settings
```

Install dependencies for both backend and frontend:

```bash
cd backend && pnpm install
cd ../frontend && pnpm install
```

## Start Development Servers

Open two terminals:

**Terminal 1 вЂ” Backend:**

```bash
cd backend
pnpm dev
```

Starts the Express API server on port 3001 with `--watch` for auto-reload.

**Terminal 2 вЂ” Frontend:**

```bash
cd frontend
pnpm dev
```

Starts the SvelteKit dev server on port 3000 with hot module replacement.

| Service  | URL                    |
|----------|------------------------|
| Frontend | http://localhost:3000   |
| Backend  | http://localhost:3001   |

The frontend proxies API calls to the backend automatically.

:::caution
HTTP mode does not set secure cookies and does not test CORS or mixed-content behavior. Use [HTTPS development mode](/permish/development/https-dev/) to catch production issues early.
:::

## Key Files

| File | Purpose |
|------|---------|
| `backend/src/index.js` | Express entry point. Route mounting order matters. |
| `backend/src/db/schema.js` | SQLite schema and migrations. Definitive field names. |
| `frontend/src/lib/api.ts` | Backend API client (fetch wrapper). |
| `frontend/src/app.css` | Theme variables (oklch colors, dark mode). |
| `frontend/src/lib/components/ui/` | shadcn-style base components. |

## Rebuilding Native Modules

After switching Node versions, rebuild the native SQLite binding:

```bash
cd backend
pnpm rebuild better-sqlite3
```
