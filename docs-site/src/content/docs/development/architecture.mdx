Permish supports two backend modes and three data storage modes. This page explains how the pieces fit together.

## Backend Modes

### Express Mode (Default)

The Express backend handles auth, CRUD, PDF generation, email, and SMS. Data is stored in SQLite via `better-sqlite3`.

```
Browser → SvelteKit Frontend → Express Backend → SQLite
```

### PocketBase Mode

PocketBase replaces Express for auth, CRUD, and realtime subscriptions. A Node.js sidecar handles PDF generation, email, and SMS since PocketBase cannot do those natively.

```
Browser → SvelteKit Frontend → PocketBase (auth, CRUD)
                             → Node Sidecar (PDF, email, SMS)
```

## Data Modes

The app supports three data storage modes, selectable via **Settings > Data Storage**:

| Mode     | Storage            | Connectivity | Description                                     |
|----------|--------------------|--------------|--------------------------------------------------|
| `online` | Server             | Required     | Default for web. Server is the source of truth.  |
| `local`  | In-browser SQLite  | None         | Fully offline. Data stays in the browser (sql.js).|
| `hybrid` | Local SQLite + sync| Optional     | Local-first with background sync to PocketBase.  |

- **online** — Default for web deployments.
- **local** — Default for the desktop Tauri app. All data is stored locally.
- **hybrid** — Useful for intermittent connectivity (e.g., camp locations with poor signal). Works offline and syncs when a connection is available.

## Key Files

| File | Purpose |
|------|---------|
| `backend/src/index.js` | Express entry point. Route mounting order is critical (form routes before event routes). |
| `backend/src/db/schema.js` | SQLite schema and migrations. The definitive source for field names. |
| `backend/src/services/pdf.js` | PDF generation using `pdf-lib`. Fills the official church PDF template. |
| `backend/src/templates/permission-form.pdf` | The actual church form used as a PDF template. |
| `frontend/src/lib/api.ts` | Backend API client. All HTTP requests go through this module. |
| `frontend/src/app.css` | Theme variables in oklch format with dark mode support. |
| `frontend/src/lib/components/ui/` | Hand-built shadcn-style base components (Svelte 5 runes). |
| `docker-compose.yml` | Docker Compose configuration with profiles for Express, PocketBase, and production HTTPS. |

## Route Mounting Order

In `backend/src/index.js`, form routes **must** be mounted before event routes since both use the `/api/events` prefix:

1. Auth routes (`/api/auth`)
2. Form routes (`/api/events` — public: `/:id/form`, `/:id/submit`)
3. Events routes (`/api/events` — planner-only, requires auth)
4. Profiles routes (`/api/profiles`)
5. Submissions routes (`/api/submissions` — `/mine` before `/:id/*`)
